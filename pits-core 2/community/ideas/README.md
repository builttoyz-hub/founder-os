# /community/ideas — Ideas de la Comunidad

## Propósito

El buzón de sugerencias de la comunidad. Cada idea que llega — por DM, comentario, WhatsApp, email, o en persona — se documenta aquí antes de que se pierda.

> **Regla importante:** Documentar una idea aquí NO significa que se va a implementar. Significa que fue escuchada y registrada. La decisión de implementar la toma el Product Owner con criterios de priorización claros.

## Proceso

1. **Capturar:** Cualquier miembro del equipo puede crear un archivo aquí cuando recibe una idea de la comunidad
2. **Tagear:** Agregar el tag de categoría al nombre del archivo
3. **Revisar:** El Product Owner revisa `/community/ideas/` semanalmente
4. **Decidir:** Las ideas que avanzan se convierten en Issues en GitHub o entran al backlog de producto
5. **Cerrar el loop:** Si una idea de un usuario específico se implementa, notificarle

## Convención de Nombres

```
idea-[categoria]-[descripcion]-[YYYY-MM-DD].md

Categorías: search | listing | trust | mobile | payments | categories | social | other

Ejemplos:
  idea-search-filtrar-por-motorización-2026-07-18.md
  idea-trust-video-de-la-pieza-2026-07-22.md
  idea-mobile-camera-photo-upload-2026-08-01.md
```

## Template Rápido

```markdown
# Idea: [Título Descriptivo]

**Fuente:** DM de Instagram / Comentario / WhatsApp / En persona / Email
**Usuario (alias si es identificable):** [Alias o "anónimo"]
**Fecha:** YYYY-MM-DD
**Categoría:** search | listing | trust | mobile | payments | other

## La Idea en Sus Palabras

"[Cita directa del usuario si está disponible, o paráfrasis fiel]"

## Entendimiento del Problema Subyacente

[¿Qué frustración o necesidad real está detrás de esta sugerencia?]

## Frecuencia

¿Cuántas veces hemos escuchado algo similar?
- [ ] Primera vez
- [ ] 2–5 veces
- [ ] Más de 5 veces — patrón claro

## Estado

- [ ] Registrada
- [ ] En evaluación
- [ ] Agregada al backlog
- [ ] Implementada (link al PR/Issue)
- [ ] Descartada (razón: ___)
```
