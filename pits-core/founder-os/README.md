# /founder-os — Sistema Operativo del Fundador

## Propósito

Esta carpeta es el sistema central de operación de Santiago Tarazona como Fundador y CEO de PITS MARKET. Contiene los marcos de trabajo, ritmos operacionales, decisiones importantes y el contexto que permite que la empresa funcione con consistencia — incluso cuando el fundador no está disponible.

> **Principio:** Un buen founder OS hace que el fundador sea un multiplicador, no un cuello de botella.

## ¿Por qué existe esta carpeta?

Cuando una empresa tiene un solo fundador:

1. **El conocimiento implícito es un riesgo.** Todo lo que vive solo en la cabeza del fundador es un punto único de falla. Documentar los marcos de trabajo los hace transferibles.

2. **La consistencia requiere sistemas.** Las decisiones ad-hoc crean inconsistencias. Los marcos de trabajo documentados crean predictibilidad.

3. **El crecimiento requiere delegación.** No se puede delegar lo que no está documentado. Cada proceso que se escribe aquí es un proceso que eventualmente puede delegar a un miembro del equipo.

4. **Los inversionistas valoran la estructura.** Un fundador con un sistema operativo visible señala madurez empresarial.
